# 学习路径

这份文档不是在讲某一个模型，而是在讲整条学习路径该怎么组织。

目标不是一次性理解所有论文细节，而是沿着一条连续的路径，从最基础的深度学习训练，一步一步走到扩散超分和工程实践。

推荐主线如下：

1. 手写数字识别
2. VAE 图片生成
3. DDPM 基础扩散
4. 超分基线
5. SR3 条件扩散超分
6. ResShift 少步扩散超分
7. 工程实践与扩展练习

## 第一阶段：手写数字识别

目标不是“刷高分”，而是先完整跑通一个深度学习项目。

### 这一阶段要掌握什么

- 样本、标签、batch 是什么
- 模型前向传播在做什么
- loss、反向传播、优化器各自承担什么职责
- 训练集和验证集为什么要分开

### 对应代码

- 配置：`configs/classification/cnn.yaml`
- 入口：`main.py`
- 基础训练器：`trainer/base.py`
- 模型：`models/cnn.py`

### 这一阶段对应的工程实践

- 如何通过配置文件控制实验
- 为什么入口文件要保持薄
- checkpoint 和实验目录的作用
- 为什么 trainer 要和 model 分开

### 可以直接做的动作

1. 先跑 `cnn.yaml`
2. 再跑 `resnet.yaml`
3. 比较两个模型在结构和参数量上的差异
4. 修改 batch size、lr、epoch，观察训练结果变化

## 第二阶段：VAE 图片生成

这一阶段的作用是把视角从“判别任务”带到“生成任务”。

### 这里要完成的认知转变

分类任务关心：

- 这张图是什么类别

VAE 关心：

- 如何把图像压缩到潜空间
- 如何从潜空间重建出图像
- 如何从潜变量中采样生成新图

### 这一阶段要掌握什么

- `mu` 和 `logvar` 的含义
- 为什么需要重参数化
- 为什么 VAE loss 由重构项和 KL 项组成
- 为什么生成任务不再用准确率评估

### 对应代码

- 配置：`configs/generate/vae.yaml`
- 模型：`models/vae.py`
- 训练器：`trainer/vae.py`
- 推理脚本：`inference_vae.py`

### 这一阶段对应的工程实践

- 一个 trainer 如何覆写基类的 loss / evaluate
- 模型输出为什么可以不只是一个张量
- 如何把训练逻辑和推理逻辑拆开

## 第三阶段：DDPM 基础扩散

这一阶段是把 VAE 的“潜空间建模”进一步推进到“过程建模”。

### 这里要完成的认知转变

VAE 更像：

- 一次编码
- 一次解码

DDPM 更像：

- 一张图逐步加噪
- 再逐步去噪恢复

### 这一阶段要掌握什么

- 时间步 `t` 为什么重要
- 正向扩散和反向去噪分别是什么
- 为什么训练目标通常是预测噪声
- 为什么扩散模型采样慢

### 对应代码

- 配置：`configs/generate/ddpm.yaml`
- UNet：`models/ddpm/unet.py`
- 调度器：`models/ddpm/diffusion.py`
- 训练器：`trainer/diffusion.py`

### 这一阶段对应的工程实践

- 同一个模型和一个 scheduler 如何协同
- 训练时的目标不一定是最终输出本身
- 指标和可视化可以是两套并行机制

## 第四阶段：超分基线

这一阶段的作用是从“无条件生成”转向“图像恢复任务”。

### 这里要完成的认知转变

以前是：

- 从图像到类别
- 或从噪声到图像

现在变成：

- 已经有一张低质量图
- 目标是把它恢复得更接近高质量图

### 这一阶段要掌握什么

- 为什么超分任务的数据不再是 `(image, label)`
- 为什么会变成 `(lr_up, hr)`
- 为什么评估指标变成 PSNR
- 为什么 SR 常常学习残差

### 对应代码

- 配置：`configs/sr/srresnet.yaml`
- 数据包装：`data/sr_dataset.py`
- 模型：`models/sr.py`
- 训练器：`trainer/sr.py`

### 这一阶段对应的工程实践

- 原始数据集和任务样本如何分离
- 指标如何随任务改变
- 训练过程中如何保存可视化结果

## 第五阶段：SR3 条件扩散超分

这是整条路径里最重要的桥梁阶段。

如果从 DDPM 直接跳到 ResShift，跨度会很大。

SR3 的作用，就是先接受下面这件事：

- 扩散模型不只能做无条件生成
- 它也可以带条件输入做图像恢复

### 这里要完成的认知转变

DDPM：

- 从噪声逐步生成图像

SR3：

- 给定低清条件图
- 逐步恢复出高清图

### 这一阶段要掌握什么

- 什么叫条件扩散
- 为什么模型输入会从 `x_t` 变成 `[x_t, condition]`
- 为什么 SR3 是 DDPM 到 ResShift 的桥梁

### 对应代码

- 配置：`configs/sr/sr3.yaml`
- 模型：`models/sr3.py`
- 训练器：`trainer/sr3.py`
- 调度器复用：`models/ddpm/diffusion.py`

### 这一阶段对应的工程实践

- 如何在不重写整套框架的前提下复用已有 UNet 和 diffusion scheduler
- 如何通过新增 trainer 和 config 扩展一个新任务
- 为什么“条件信息”通常通过拼接通道注入模型

## 第六阶段：ResShift 少步扩散超分

这一阶段不是重新介绍一遍扩散，而是强调“为什么在恢复任务里要做更高效的改造”。

### 这里要完成的认知转变

SR3 已经说明：

- 条件扩散可以做超分

ResShift 进一步说明：

- 在恢复任务里没必要完全照搬无条件扩散
- 可以围绕残差和少步采样做更贴近任务的设计

### 这一阶段要掌握什么

- 为什么恢复任务里条件图已经包含大量结构信息
- 为什么 ResShift 更关注残差而不是整图生成
- 为什么它的采样步数可以更短

### 对应代码

- 配置：`configs/sr/resshift.yaml`
- 模型与调度器：`models/resshift.py`
- 训练器：`trainer/resshift.py`

### 这一阶段对应的工程实践

- 怎样在相同框架里继续迭代算法
- 怎样把“研究动机”落到工程实现上
- 怎样比较不同方案的指标、速度和复杂度

### 这一阶段建议补做的内容

- 先对比 `SR3` 和 `ResShift` 的输入、输出、loss 与采样起点
- 再从 `trainer/sr3.py` 出发，列出改造成 `trainer/resshift.py` 的最小改动
- 最后手推 `x_t = LR_up + s_t * (HR - LR_up) + sigma_t * z` 这类 shifted 公式

建议把 `docs/resshift.md` 当成这一阶段的主文档，它现在已经补上了：

- `SR3 vs ResShift` 的结构化对比
- 从 `SR3` 改到 `ResShift` 的实现路线
- 论文核心公式的复现题
- 一套不带答案的练习题

## 第七阶段：工程实践与扩展

前六个阶段学的是“从模型到任务”。

最后这一阶段要真正开始上手“做项目”。

### 应该强化的工程实践主题

- 配置驱动训练
- trainer / model / data 分层
- 注册表模式
- checkpoint 管理
- 指标与可视化
- 随机种子与复现
- 任务扩展时如何复用已有代码

### 可以继续做的练习

1. 给分类任务增加 `FashionMNIST` 配置。
2. 给 VAE 增加不同 latent dim 的实验。
3. 给 DDPM 改不同的噪声调度策略。
4. 给 SR3 改不同的扩散步数，对比效果和速度。
5. 对比 SRResNet、SR3、ResShift 的可视化结果和 PSNR。
6. 尝试把超分数据从 `MNIST` 替换到 `CIFAR10`。

## 组织方式

如果按整条路径来排，比较自然的顺序是：

### 第一部分：深度学习入门

- CNN / ResNet
- 标准训练流程
- 配置和实验管理

### 第二部分：生成模型入门

- VAE
- 潜变量和生成

### 第三部分：扩散模型入门

- DDPM
- 时间步和噪声预测

### 第四部分：图像恢复任务

- 超分基线
- 数据包装和 PSNR

### 第五部分：扩散超分

- SR3
- ResShift

### 第六部分：工程实践

- 重构
- 扩展
- 对比实验
- 写自己的 trainer / model / config

## 一句总纲

如果要把整条路径压缩成一句话，可以直接概括成：

先用手写数字识别学会“怎么训练一个模型”，再用 VAE 学会“怎么生成图像”，再用 DDPM 学会“怎么逐步建模生成过程”，然后用 SR3 和 ResShift 学会“怎么把扩散思想用到条件超分和工程实现里”。
