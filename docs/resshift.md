# ResShift 学习说明

这份文档不只解释 “ResShift 是什么”，还重点回答下面四个问题：

1. 它和 `SR3` 到底差在哪。
2. 它的输入输出、训练目标、推理过程分别改了什么。
3. 如果你已经有一个 `SR3`，应该怎么把它一步步改成 `ResShift`。
4. 论文里的核心公式，哪些值得自己手推和复现。

如果你还没有读过 `DDPM` 和 `SR3`，建议先看 `docs/generative_basics.md` 与 `docs/learning_path.md`，再回到这里。

## 为什么在 SR3 之后学 ResShift

`SR3` 已经证明一件事：

- 扩散模型可以带条件做超分

但 `ResShift` 继续追问另一件事：

- 对恢复任务来说，真的有必要像无条件生成那样，从纯噪声慢慢采样很多步吗

对于超分、去噪、去模糊这类任务，低质量输入本身已经包含大量结构信息。`ResShift` 的核心想法，就是不要再把问题表述成“从纯噪声生成整张图”，而要表述成“围绕条件图逐步恢复目标残差”。

这也是它和 `SR3` 在学习路径里的分工：

- `SR3` 负责让你理解条件扩散怎么成立
- `ResShift` 负责让你理解条件扩散怎样进一步贴近恢复任务

## 先给出一句定义

`ResShift` 可以理解成：

- 给定退化图 `y`
- 定义目标残差 `r = x_0 - y`
- 在时间步里逐步恢复这个残差
- 再由 `x = y + r` 重建高清图

本仓库里的简化版实现使用的是上采样后的低清图 `LR_up` 作为条件，因此残差写成：

`R = HR - LR_up`

## 和 SR3 的核心对比

### 1. 问题表述不同

`SR3` 的典型表述是：

- 条件输入是低清图 `LR_up`
- 当前状态是被加噪后的高清图 `x_t`
- 模型学习预测噪声 `epsilon`

可以写成：

`epsilon_theta(x_t, LR_up, t)`

`ResShift` 的典型表述是：

- 条件输入仍然是 `LR_up`
- 当前状态不再是“从高清图整图加噪得到的样本”
- 而是围绕残差 `R = HR - LR_up` 构造一个 shifting 过程
- 模型更直接地学习目标残差 `R`

可以写成：

`R_theta(x_t, LR_up, t)`

### 2. 输入输出不同

把两者放在一张表里更清楚：

| 项目 | SR3 | ResShift |
| --- | --- | --- |
| 条件输入 | `LR_up` | `LR_up` |
| 训练状态 `x_t` | `HR` 经扩散加噪得到的样本 | 条件图与目标残差混合后的 shifted 样本 |
| 网络输入 | `[x_t, LR_up], t` | `[x_t, LR_up], t` |
| 网络输出 | 预测噪声 `epsilon` | 预测残差 `R = HR - LR_up` |
| 最终重建 | 通过去噪公式逐步恢复 `HR` | 通过 `LR_up + predicted_residual` 重建 `HR` |

这里最容易被忽略的一点是：

- 两者的网络骨架可以几乎一样
- 但监督目标和采样更新公式已经不是一回事

### 3. 推理起点不同

`SR3` 常见推理起点是：

- 从纯高斯噪声开始
- 每一步结合条件图去噪

本仓库里的 `ResShift` 则是：

- 从条件图附近的一个带噪状态开始
- 每一步预测当前应该补回多少残差

所以它更像“围绕条件图修图”，而不是“从纯噪声生图”。

### 4. 少步采样的动机不同

`SR3` 延续的是标准扩散范式，通常时间步较多。

`ResShift` 的动机是：

- 恢复任务已有条件输入
- 目标不需要从零生成
- 因此可以把采样路径设计得更短

这就是论文和官方仓库都强调的高效推理背景。

## 本仓库里的 ResShift 在做什么

这里实现的是简化版 `ResShift`，目标不是一比一复现论文，而是把下面这条链路讲清楚：

1. 从数据加载器拿到 `(LR_up, HR)`。
2. 定义残差 `R = HR - LR_up`。
3. 在不同时间步上，构造一个带残差比例和噪声比例的 shifted 状态。
4. 让网络看到 `([x_t, LR_up], t)` 之后，直接预测 `R`。
5. 采样时用少量 step 逐步恢复残差，再得到最终高分图。

对应代码：

- 模型与调度器：`models/resshift.py`
- 训练器：`trainer/resshift.py`
- 配置：`configs/sr/resshift.yaml`

## 代码视角下，SR3 到 ResShift 改了什么

### 第一步：训练目标从噪声预测改成残差预测

在 `SR3` 中，训练器会做：

1. 从 `HR` 采样噪声 `epsilon`
2. 构造 `x_t`
3. 让模型预测 `epsilon`
4. 用 `MSE(predicted_noise, epsilon)` 训练

在本仓库的 `ResShift` 中，训练器改成：

1. 先计算 `R = HR - LR_up`
2. 构造 shifted 样本 `x_t`
3. 让模型预测 `R`
4. 用 `MSE(predicted_residual, R)` 训练

这是第一处最本质的改变，因为它决定了模型“到底在学什么”。

### 第二步：前向过程从整图扩散改成条件图附近的残差迁移

这版 `ResShift` 的 forward 过程可以写成：

`x_t = LR_up + s_t * R + sigma_t * z`

其中：

- `R = HR - LR_up`
- `s_t` 表示当前保留多少目标残差
- `sigma_t` 表示当前注入多少噪声
- `z ~ N(0, I)`

这和 `SR3` 的标准扩散式写法不同。`SR3` 更像在整张目标图上逐步加噪；这里更像从条件图出发，控制“残差保留比例 + 噪声强度”。

### 第三步：采样更新从 DDPM 去噪公式改成残差重建

在 `SR3` 里，推理时通常会：

- 当前有 `x_t`
- 模型输出 `predicted_noise`
- 用扩散调度器计算 `x_{t-1}`

在本仓库这版 `ResShift` 里，推理时会：

- 当前有 `current`
- 模型输出 `predicted_residual`
- 用上一个时间步对应的残差比例和噪声比例，重建新的 `current`

当 `t = 0` 时，直接输出：

`HR_hat = LR_up + predicted_residual`

### 第四步：网络主体可以几乎不变

这里最值得注意的一点是：

- `SR3UNet` 和 `ResShiftUNet` 在本仓库里都复用了同一个扩散 UNet 骨架
- 输入形式仍然是双通道拼接 `([x_t, condition], t)`
- 主要变化发生在 trainer 和 scheduler

也就是说，从工程实现角度看：

- 不是先推翻网络
- 而是先修改任务定义、监督目标和状态转移

## 从 SR3 改造成 ResShift 的建议路线

### 改造 1：先只改训练目标

目标：

- 把 `SR3` 的“预测噪声”改成“预测残差”

具体动作：

1. 保留网络输入形式 `[x_t, LR_up]`
2. 新增 `residual = HR - LR_up`
3. 暂时不改网络骨架
4. 把 loss 改成 `MSE(predicted_residual, residual)`

这一阶段最关键的是区分“监督目标”和“模型结构”是两层不同问题。

### 改造 2：再改前向过程

目标：

- 不再直接沿用 `SR3` 的 `q_sample(HR, t, noise)`
- 改成围绕条件图的 shifted 过程

可以先自己实现：

`x_t = LR_up + s_t * (HR - LR_up) + sigma_t * z`

然后回答两个问题：

1. 当 `s_t = 1`、`sigma_t = 0` 时，`x_t` 是什么。
2. 当 `s_t = 0`、噪声较大时，`x_t` 更接近什么。

### 改造 3：最后改推理循环

目标：

- 不再通过标准 DDPM 公式从 `predicted_noise` 还原 `x_{t-1}`
- 改成通过 `predicted_residual` 逐步更新当前状态

可以先写最简版本：

`x_{t-1} = LR_up + s_{t-1} * predicted_residual + sigma_{t-1} * z`

然后再思考：

- 如果要更贴近论文，还能怎样设计残差迁移路径

### 改造 4：最后才讨论“为什么能少步”

不少人一开始就会问：

- 既然 `ResShift` 更快，是不是只要把 step 改少就行

这正好是一个要纠正的误区。

正确顺序应该是：

1. 先改问题定义
2. 再改训练目标
3. 再改 forward / reverse 过程
4. 最后再讨论“少步采样为什么合理”

否则会把 `ResShift` 误解成“只是缩短版 SR3”。

## 论文公式，哪些值得自己复现

这里不要求一次性抄完整篇论文，而是建议优先复现下面几类公式。

### 公式 1：残差定义

先写清楚目标残差：

`r = x_0 - y`

在本仓库里，符号可以替换成：

`R = HR - LR_up`

这是所有后续推导的起点。

### 公式 2：shifted 状态构造

这版实现里最值得复现的是这一类：

`x_t = y + s_t * r + sigma_t * z`

你至少要能回答：

1. 这个式子里哪些量是已知的。
2. 哪些量是采样得到的。
3. 为什么它比“直接对整张 HR 做扩散”更贴近恢复任务。

### 公式 3：模型预测目标

把训练目标写成：

`L = E[||f_theta(x_t, y, t) - r||^2]`

这一步的重点不是记公式，而是确认：

- 模型输出已经从 `epsilon` 变成了 `r`

### 公式 4：最终重建

当模型给出 `r_hat` 后，最终结果是：

`x_hat = y + r_hat`

这一步很适合自己在代码里 trace：

- `condition`
- `predicted_residual`
- `current`

看看最后一张图到底是怎样从这三者拼出来的。

## 自查清单

学完这部分后，至少要能独立说清下面几句：

- `SR3` 的输出是噪声预测，`ResShift` 的输出是残差预测。
- `SR3` 更像标准条件扩散，`ResShift` 更像面向恢复任务的残差迁移。
- 从工程实现看，`ResShift` 主要改 trainer 和 scheduler，不一定先改 UNet 骨架。
- `ResShift` 的少步采样来自任务表述变化，不只是“粗暴减少 step 数”。

## 这版实现和官方版的差异

仍然要注意：本仓库是简化实现，不是官方完整版复现。

主要差异包括：

- 数据集使用 `MNIST / FashionMNIST / CIFAR10`，不是高分辨率自然图像恢复基准。
- 退化流程主要是下采样与插值回原尺寸，没有完整真实退化链。
- 网络规模较小，方便本地快速训练和阅读代码。
- 这里更强调“残差迁移 + 少步采样”这条主线，而不是追求论文指标。

## 延伸建议

如果后面要继续往下带，可以安排这三类扩展题：

1. 把 `linear` residual schedule 换成别的 schedule，观察恢复稳定性。
2. 把 `ResShift` 的监督目标从“直接残差”扩展成“残差参数化的其他形式”。
3. 对比 `SRResNet`、`SR3`、`ResShift` 的推理时延和视觉误差模式。
