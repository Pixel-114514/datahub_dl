# ResShift 学习说明

## 先说结论

ResShift 适合放进培训项目，不是因为它“更复杂”，而是因为它把扩散式图像恢复里最值得讲的一个工程问题说得很直接：

- 传统扩散恢复模型效果强，但推理步数通常很多
- 对超分、去噪、去 JPEG 等恢复任务来说，输入本来就已经带有大量结构信息
- 因此没必要像无条件生成那样，从纯高斯噪声开始慢慢采样

ResShift 的核心思路，就是围绕“退化图像到目标图像之间的残差”去设计一个更短、更贴近恢复任务本质的迁移过程。

## 官方方法的核心点

参考论文和官方仓库，ResShift 主要强调下面几件事：

- 它把图像恢复看成一个 residual shifting 过程，而不是标准的从纯噪声生成图像
- 条件输入是退化图像，模型关注的是如何逐步把目标残差恢复出来
- 相比传统扩散恢复方法，它希望用更少的采样步数完成推理
- 官方项目把它用于 super-resolution、deblurring、de-noising、face restoration 等多类恢复任务

论文和官方仓库都强调了它的高效推理特性，常见设置会把采样步数控制在 15 步这一量级。

参考：

- 论文：<https://arxiv.org/abs/2307.12348>
- 官方仓库：<https://github.com/zsyOAOA/ResShift>

## 本仓库里的实现是什么

这里实现的是一个教学版 `ResShift`，目标不是复现论文全部细节，而是把下面这条链路讲清楚：

1. 先构造低清图 `LR_up`
2. 计算高分图和低清图之间的残差 `R = HR - LR_up`
3. 在不同时间步里，对这个残差施加不同强度的保留和噪声扰动
4. 让网络学习：给定当前状态 `x_t` 和条件图 `LR_up`，预测残差 `R`
5. 推理时用少量 step 逐步恢复残差，再重建高分结果

对应代码位置：

- 调度器：`models/resshift.py`
- 网络：`models/resshift.py`
- 训练器：`trainer/resshift.py`
- 配置：`configs/sr/resshift.yaml`

## 这版教学实现和官方版的差异

差异必须讲清楚，否则培训时容易误导学员。

### 1. 数据集差异

官方方法面向真实图像恢复任务，常用高分辨率自然图像数据。

本仓库为了教学和快速跑通，默认使用：

- `MNIST`
- `FashionMNIST`
- `CIFAR10`

并通过程序内下采样构造超分样本。

### 2. 退化模型差异

官方超分/恢复任务会使用更复杂、更真实的退化流程。

本仓库默认只做：

- 下采样
- 插值回原尺寸
- 可选少量噪声

这是为了先讲清楚训练接口和条件恢复逻辑。

### 3. 网络规模差异

官方实现包含更强的网络设计和训练配置。

本仓库使用的是小型 UNet，适合：

- 本地教学
- CPU / 单卡演示
- 快速理解代码路径

### 4. 目标差异

官方项目追求真实恢复效果和速度表现。

本仓库更看重：

- 学员能否理解残差迁移
- 学员能否读懂 trainer / model / scheduler 的配合
- 学员能否自己继续扩展到真实 SR 数据集

## 建议怎么讲给学员

推荐按下面顺序：

1. 先讲 `SRResNet`：输入 bicubic 图，直接学残差
2. 再讲 `ResShift`：不是一次性回归，而是少步数逐步恢复残差
3. 最后和 `DDPM` 对比：一个偏无条件生成，一个偏条件恢复

最重要的一句可以直接给学员：

> SRResNet 是“一步修图”，ResShift 是“带时间步的多步残差修图”。

## 下一步扩展建议

如果你后面想把它进一步做成更完整的培训项目，建议继续补这几项：

- 增加 `DIV2K / Set5 / Set14` 这类真实 SR 数据支持
- 把退化流程从单纯 bicubic 扩展到 blur + resize + noise + jpeg
- 增加 LPIPS、SSIM 等指标
- 增加单独的超分推理脚本
- 把教学版 ResShift 再往官方实现靠近
